# fzfx bugs — 2026-04-01

## Hung process on C-l navigate up

- `become:fzfx --navigate up {} {q}` passes the full fzf line including tab-delimited status column
- Process tree: parent fzfx (stopped, Tl+) with child `fzfx --navigate up  ?24-bit-truecolor.sh`
- The `\t` delimiter from fd output was interpreted as a tab, `?` git status prefix leaked through
- Both processes stuck in `do_sig` (stopped signal), unresponsive to C-c, needed kill -9
- Need to investigate: why does cmdNavigate hang instead of erroring?

## cmdOpen not receiving cwd

- When navigating into a subdirectory then opening a file with alt-enter, the cwd context may be lost
- cmdOpen uses `withCfg` which sets cwd from config, but after navigation the config might not reflect the current directory
- Need to verify: does cmdEdit (renamed from cmdOpen) get the right working directory after C-o navigation?
